## Overview

Add pages to an existing Notion Database along with its properties and contents of the page itself. This block is particularly beneficial for efficiently managing text-heavy workflows, such as content editing workflows.

<Note>
  To run this block, you will first need to authenticate your Notion account
  with Keyflow. Navigate to your
  [credentials](https://keyflow.space/settings/credentials) page and connect
  your Notion.
</Note>

## Block Inputs

| Name              | Type   | Description                                                                                                                                                                                           |
| ----------------- | ------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `database_id`     | string | The URL or unique identifier of the Notion database where the data will be written.                                                                                                                   |
| `properties_json` | object | A JSON object specifying all available fields in the Notion database. This can be retrieved using the [Extract Notion Database Properties Block](/blocks/loaders/extract-notion-database-properties). |
| `values`          | object | Represents the values to be written to each column in the Notion database, matching the type specified in the database's schema.                                                                      |

## Block Outputs

| Name      | Type   | Description                                               |
| --------- | ------ | --------------------------------------------------------- |
| `page_id` | string | The page ID of the new Notion page added to the database. |

## Use Cases

1. **Automatically Logging Data:** Organize or analyze data by automatically logging it into a Notion database.
2. **Capturing Form Responses:** Record form responses directly into a Notion database for easy tracking and analysis.
3. **Synchronizing Data:** Ensure seamless data synchronization between Notion and other platforms by automatically adding new entries whenever updates are detected elsewhere.
4. **Adaptable to Various Use Cases:** Use it in any scenario where data entry into Notion from an autonomous workflow is required.
