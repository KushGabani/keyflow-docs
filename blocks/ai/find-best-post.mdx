## Overview

Identify the best post that aligns closely with a given query to aid users in locating the most relevant content quickly.

## Block Inputs

| Name             | Type      | Description                                                                                                   |
| ---------------- | --------- | ------------------------------------------------------------------------------------------------------------- |
| `query`          | string    | The question/topic regarding which the post needs to be found.                                                |
| `titles`         | string[ ] | A list of post titles to consider when searching for the best post.                                           |
| `links`          | string[ ] | Corresponding to the list of titles, this is an array of URLs for the respective posts.                       |
| `model`          | selector  | LLM Model to be used for inference. Can be one of the following supported models.                             |
| `cache_response` | boolean   | When enabled, it saves you AI calls by returning the same response if the inputs to the LLM have not changed. |

## Block Outputs

| Name            | Type   | Description                                                     |
| --------------- | ------ | --------------------------------------------------------------- |
| `best_post_url` | string | The URL of the top post that most accurately matches the query. |

## Use Cases

1. **Navigate Through Content:** Easily find a post that best fits a specific query or topic among a large volume of content.
2. **Content Curation:** Streamline the process of content curation by providing users with the most relevant links based on their interests or search terms.
3. **Enhance User Experience:** Improve user experience by quickly directing them to the most pertinent content without manually searching through multiple posts.
