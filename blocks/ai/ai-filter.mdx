## Overview

Processes data by applying a natural language condition to each input.

## Block Inputs

| Name             | Type     | Description                                                                                                   |
| ---------------- | -------- | ------------------------------------------------------------------------------------------------------------- |
| `input`          | string   | The text to apply the filter on.                                                                              |
| `condition`      | string   | The natural language condition you want to be applied to the inputs.                                          |
| `model`          | selector | LLM Model to be used for inference. Can be one of the following supported models.                             |
| `cache_response` | boolean  | When enabled, it saves you AI calls by returning the same response if the inputs to the LLM have not changed. |

## Block Outputs

| Name              | Type   | Description                                                                                                                                                                                  |
| ----------------- | ------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `filtered_output` | string | The result of the AI Filter's evaluation. If the input data meets the specified condition, you will receive the same input data here. If it does not meet the condition, this will be empty. |

## Use Cases

1. **Filtering Customer Inquiries:** Identifying and separating inquiries based on specific criteria expressed in natural language.
2. **Data Categorization:** Sorting textual data according to specified conditions.
3. **Automated Decision Making:** Making dynamic decisions based on textual content in complex automation processes.
