## Output Node

### Quick Overview

- **What It Does:** Compiles the provided inputs into a named output that can be used elsewhere.
- **Useful For:** Clearly labeling and passing data from one stage of an automation to another.

### Inputs and Outputs

| Name                           | Type   | Description                                                                                                         |
| ------------------------------ | ------ | ------------------------------------------------------------------------------------------------------------------- |
| #output (any)                  | Any    | The value or data that you want to name and pass on as the output.                                                  |
| output_name (string, optional) | String | The label you assign to the output. If not provided, the output will still be recorded but without a specific name. |

### Use Cases

- **Data Transfer:** Use it to pass data from one stage of automation to another.
- **Returning Results:** Enables returning results to a triggering agent.
- **Complex Automations:** Handy in complex automations where tracking and utilizing specific pieces of data is crucial.
- **Data Processing:** Use it for results from data processing and calculation outputs.
