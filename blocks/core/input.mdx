## Overview

It allows setting a default value or accepting user input at runtime, which can be used further down the automation flow.
The input block allows setting accepting inputs into your flows. While building your automation, you can provide a test value to mock a incoming inputs.

## Block Inputs

| Name         | Type     | Description                                                                                                                     |
| ------------ | -------- | ------------------------------------------------------------------------------------------------------------------------------- |
| `input_name` | string   | A name or label of the input, This name will identify the input in user view, external flows, share as form and app.            |
| `test_value` | string   | A default value to mock the input when running a flow manually.                                                                 |
| `input_type` | selector | Specifies the type of input field in user view and form/app view. It can be either `text`, `number`, `boolean`,`file`, `audio`. |

## Block Outputs

| Name            | Type   | Description                                                               |
| --------------- | ------ | ------------------------------------------------------------------------- |
| `{input_value}` | string | The input value provided to the flow. Defaults to test_value of the block |

<Note>
  The name of the output handle `{input_value}` changes dynamically based on the
  `input_name`. So if the `input_name` is `gender`, The output handle would also
  be `gender` representing the incoming value of the input block.
</Note>

## Use Cases

1. **Provide a pre-defined default value:** Provide a default value to the flow that will be used unless a value is being input from outside.
2. **Dynamic, Genertic and Reusable automations:** Build flows that can be customized on-the-fly based on the user's input or use existing flows as blocks using external flow blocks.
3. **Form inputs and User View:** Show appropriate input fields for the flow in form view or user view.
